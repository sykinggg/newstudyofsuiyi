Lifecycle Hooks(组件的生命周期)

Component lifecycle hooks overview(概述)

描述
每个接口都有一个单一的钩子方法，其名称是以ng为前缀的接口名称
例如，OnInit接口有一个名为ngOnInit（）的钩子方法，Angular在创建组件后立即调用
        export class PeekABoo implements OnInit {
            constructor(private logger: LoggerService) { }
          
            // implement OnInit's `ngOnInit` method
            ngOnInit() { this.logIt(`OnInit`); }
          
            logIt(msg: string) {
              this.logger.log(`#${nextId++} ${msg}`);
            }
        }
没有指令或组件将实现所有的生命周期钩子

Lifecycle sequence(生命周期序列)

通过调用其构造函数创建组件/指令后，Angular在特定时刻按以下顺序调用生命周期钩子方法
Hook Purpose and Timing(目的和执行顺序)
ngOnChanges()
Angular响应（重新）设置数据绑定的输入属性
该方法接收当前和以前的属性值的SimpleChanges对象。
Called before ngOnInit() and whenever one or more data-bound input properties change.
在ngOnInit（）之前调用并且每当有一个或多个数据绑定输入属性发生变化时调用。
ngOnInit()
在Angular之后初始化指令/组件首先显示数据绑定属性并设置指令/组件的输入属性。
Called once, after the first ngOnChanges().
在第一个ngOnChanges（）之后调用一次。
ngDoCheck()
检测Angular无法或无法自行检测到的更改并采取相应措施。
Called during every change detection run, immediately after ngOnChanges() and ngOnInit()
在每次更改检测运行期间，在ngOnChanges（）和ngOnInit（）之后立即调用
ngAfterContentInit()
Respond after Angular projects external content into the component's view.
在Angular将外部内容投影到组件视图之后作出响应。
Called once after the first ngDoCheck().
在第一次ngDoCheck（）之后调用一次。
A component-only hook.
只有组件的钩子。
ngAfterContentChecked()
Respond after Angular checks the content projected into the component.
在Angular检查投影到组件中的内容之后作出响应。
Called after the ngAfterContentInit() and every subsequent ngDoCheck().
在ngAfterContentInit（）和每个后续的ngDoCheck（）之后调用。
A component-only hook.
只有组件的钩子。
ngAfterViewInit()
Respond after Angular initializes the component's views and child views.
在Angular之后进行响应，初始化组件的视图和子视图。
Called once after the first ngAfterContentChecked().
在第一个ngAfterContentChecked（）之后调用一次。
A component-only hook.
只有组件的钩子。
ngAfterViewChecked()
Respond after Angular checks the component's views and child views.
在Angular检查组件的视图和子视图之后作出响应。
Called after the ngAfterViewInit and every subsequent ngAfterContentChecked().
在ngAfterViewInit和每个后续的ngAfterContentChecked（）之后调用。
A component-only hook.
只有组件的钩子。
ngOnDestroy
Cleanup just before Angular destroys the directive/component.
在Angular破坏指令/组件之前进行清理
Unsubscribe Observables and detach event handlers to avoid memory leaks.
取消订阅Observables和分离事件处理程序以避免内存泄漏。
Called just before Angular destroys the directive/component.
在Angular破坏指令/组件之前调用。
