routes——细节 知识点

路由引用
/** * 账单日汇总月汇总controller * author: Lulin * date: 2016年12月21日 */
import { RouterModule, Routes } from '@angular/router';
实例
            const appRoutes: Routes = [
                {  
                    path: 'crisis-center/:id', 
                    component: CrisisListComponent 
                    data: { title: 'Heroes List' }
                }
                { path: '**', component: PageNotFoundComponent }
            ]

            @NgModule({
                imports: [
                    RouterModule.forRoot(
                        appRoutes,
                        { enableTracing: true } //仅用于调试
                    )
                ]
            })
描述：
appRoutes路由数组描述了如何路由,RouterModule.forRoot方法以配置路由器
路由器解析并为您建立最终的URL，允许您在应用程序视图之间路由时使用相对路径和绝对路径
data属性可以在每个激活的路由中访问。 使用它来存储项目，如页面标题，面包屑文本和其他只读的静态数据
最后一条路径中的**路径是通配符。 如果请求的URL不匹配配置中前面定义的路由的任何路径，路由器将选择此路由。 这对于显示“404 - 未找到”页面或重定向到另一个路线很有用。
路由匹配是按顺序匹配的从上到下
如果您需要查看路由生命周期中发生的事件，那么enableTracing选项将作为路由器默认配置的一部分。 这将在每个路由生命周期中发生的每个路由器事件输出到浏览器控制台。 这应该只用于调试目的。 在作为RouterModule.forRoot（）方法的第二个参数传递的对象中设置enableTracing：true选项。

router links——细节 知识点

template 链接跳转
            template: `
            <h1>Angular Router</h1>
                <nav>
                    <a routerLink="/crisis-center" routerLinkActive="active">Crisis Center</a>
                    <a routerLink="/heroes" routerLinkActive="active">Heroes</a>
                </nav>
            <router-outlet></router-outlet>
            `
描述
使用routerLink来进行路由跳转
RouterLinkActive指令有助于在视觉上区分当前选定的“活动”路由的锚点 当相关的RouterLink激活时，路由器将激活的CSS类添加到元素

router state——细节 知识点

描述
在每个成功的路由生命周期结束之后，路由器将构建构成路由器当前状态的ActivatedRoute对象树。 可以使用路由器服务和routerState属性从应用程序中的任何位置访问当前的RouterState。

activeted route——细节 知识点

描述
路由路径和参数可以通过一个名为ActivatedRoute的注入路由器服务来获得
Property Description
url 路径路径的可观察性，表示为路径路径每个部分的字符串数组
data 包含为路由提供的数据对象 包含解析警卫的任何已解决的值
paramMap 包含特定于路由的必需和可选参数的映射 支持从相同的参数中检索单个和多个值
queryParamMap 包含可用于所有路由的查询参数的映射的Observable 映射支持从查询参数中检索单个和多个值
fragment 所有路线可用的URL片段的可视化
outlet 用于呈现路由的RouterOutlet的名称
routeConfig 包含原始路径的路由的路由配置
parent 此路线是子路线时，路线的父级ActivatedRoute
firstChild 包含此路由的子路由列表中的第一个ActivatedRoute
children 包含当前路线下激活的所有子路线
注意
两个较旧的属性仍然可用
params
包含特定于路由的必需参数和可选参数的Observable。 改用paramMap
queryParams
包含可用于所有路由的查询参数的Observable。 改用queryParamMap

总结

Router Part Meaning
Router 显示活动URL的应用程序组件 管理从一个组件到下一个组件的路由
RouterModule 一个单独的NgModule 提供必要的服务提供者和指令来浏览应用程序视图
Routes 定义一个路由数组，每个路由将一个URL路径映射到一个组件
Route 定义路由器如何根据URL模式路由到组件 大多数路由由路径和组件类型组成
RouterOutlet 指示路由器显示视图的指令（<router-outlet>）
RouterLink 将可点击的HTML元素绑定到路由的指令 单击具有绑定到字符串或链接参数数组的routerLink指令的元素将触发路由
RouterLinkActive 当包含在元素上或内部的相关联的routerLink变为活动/不活动时， 用于从HTML元素添加/移除类的指令
ActivatedRoute 提供给每个路由组件的服务， 包含路由特定信息，如路由参数，静态数据，解析数据，全局查询参数和全局片段
RouterState 路由器的当前状态包括当前激活的路由树以及遍历路由树的简便方法
Link parameters array 路由器解释为路由指令的数组 可以将该数组绑定到RouterLink，或者将该数组作为参数传递给Router.navigate方法
Routing component 一个带有RouterOutlet的角度组件，用于显示基于路由器路由的视图
